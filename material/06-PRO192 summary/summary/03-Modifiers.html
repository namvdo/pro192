Chapter 3. Modifiers

A. Notion Summary
Access Modifiers
public

·         the most generous

·         the only access modifier permitted to top-level classes

·         public method can be overridden by any subclass.

private

•         least generous

·         may not be used by a top level class

  • A private method or variable may be used only by an instance of the class that declares the variable or method.

protected

  • the more generous than private
  • only variables and methods can be declared protected
  • available to all members of the package and to subclasses
  • Subclasses:
  • may override the protected methods of the superclass
  • may read/write protected fields and call protected members of the superclass; but not protected fields and members from other instances

default

  • not accessible outside the package
  • accessible and overrideable by any class in the same package as the class in question.

The only access modifier permittted to non-nested classes is public;there is no such thing as a protected or private top-level class.
There can be at most one access modifier to an entity.
Access modifiers dictate which classes, not which instances, may access features.
Methods may not be overridden to be more private.
legal overridden method access:
private to default to protected to public

Other Modifiers
final

  • applies to classes, variables and method
  • final features may not be changed.
  • a final class may not be subclassed
  • a final variable may not be modified once it has been assigned a value.

abstract

  • can be applied to classes and methods
  • an abstract class cannot be instantiated
  • abstract methods are implemented by subclasses.

static

  • applies to variables, methods and code blocks
  • static variable: only one instance of the variable, even if there are many instance of the class
  • static method:

–         cannot call non-static features of their class;

–         may be invoked before a class instance is created

–         there is no "this" for static methods.

–         if overridden, must keep the static modifier

  • static initializers: at class load time, all static intializers and static code blocks are executed once.
  • any nonprivate feature may be statically imported into a source file of the same package.
  • only a public feature may may be statically imported into a source file of a different package.

native

  • refers only to methods: means the implementation is elsewhere, outside the Java Virtual Machine.

transient

  • applies only to variables
  • means its not part of an object's persistent state.

volatile

  • only to variables
  • may be modified asynchronously

synchronized

  • used to access critical code in multithreaded programs

B. Summary based on questions in the text book
1.     There are many modifier types in Java.
order of access modifiers from most restrictive to least restrictive:
        private (only in the class), default (no access modifier key word, accessable inside the package), protected (accessable inside the package or in the subclass outside the package), public (accessable every where).
2.     Legal declarations: transient int i = 41; public final static native int foo();
abstract class A  {abstract void foo();  final void fun() {} void test() {} }
Illegal declarations: default String s; abstract double d; abstract final double hyperbolicCosine();
abstract class A  {abstract void foo()... }    class B extends A {...} (must be:  class B extends A {void foo() {}...}
class A {final void foo() {}}      class B extends A {void foo() {}} // final method cannot be overriden
3.     Abstract class:
        An abstract class cannot be directly instantiated in a client class.
An abstract class can (but is not required to) have abstract methods, and non-abstract methods.
The abstract class will eventually be subclassed, and the subclass must overrid all the parent’s abstract method(s)  or be declared abstract itself..
4.     Only variables may be transient, not methods or classes. Transient variables are not serialized.
5.     Unlike C++, in Java static modifier can be used for class level variables only, thus the following declaration is illegal: public void amethod(){static int i;}
6.     static methods do not have access to the implicit variable called this
A static method may be called without creating an instance of its class
A static method may not be overriden to be non-static
7.     Multiple static initializers are legal. All static initializer code is executed at class-load time, so before main() is ever run. E.g. when running the program:
class Main
{static int x = 25;
{ x += 20; }
static { x += 5; }
public static void main(String args[])
{System.out.println("x = " + x);
Main t = new Main();
System.out.println("t.x = " + t.x);
}
static { x /= 5; }
}
The output is
x = 6
t.x = 26

If the superclass is in a different package, the superclass version must be public or protected. (Default methods may be overridden only if the subclass is in the same package as the superclass; private methods may not be overridden at all.)
10.  "transient" modifier should be used to denote a variable that should not be written out as part of its class’s persistent state. “volatile”  modifier is used for certain multi-threaded situations.
11.  If a class does not provide implementations for all methods of all interfaces that the class declares it implements, that class must be declared abstract. Abstract classes may not be instantiated.
12.  Java’s final keyword has slightly different meanings depending on the context, but in general it says “This cannot be changed”. Classes, data, and methods can be declared final: final classes cannot be extended, final variable cannot be changed, and final method cannot be overriden.
13.  A non-inner class may not be static.
14.  An object that inherits a protected method from a superclass in a different package may call that method on itself but not on other instances of the same class. Thus the following statement is true:
If class Y extends class X, the two classes are in different packages, and class X has a protected method called abby(), then any instance of Y may call the abby() method on itself  but not on other instance of  Y.
15.  Abstract classes can contain methods that are defined and methods that are not defined.
16.  Select the order of access modifiers from least restrictive to most restrictive.
public, protected, default, private
17.  Which access modifier allows you to access method calls in libraries not created in Java?
native
        The native modifier is an indicator to the Java Virtual Machine that the method actually lives in a library outside of Java. The System.loadLibrary() method is required to indicate which library contains the method. See Chapter 3 for more information.
18.  A final object cannot be reassigned a new address in memory.
19.  Given the following code, what will be the outcome?
public class Funcs extends java.lang.Math {
public int add(int x, int y) {
return x + y;
}
public int sub(int x, int y) {
return x - y;
}
public static void main(String [] a) {
Funcs f = new Funcs();
System.out.println("" + f.add(1, 2));
}
}
The code does not compile.
        The code does not compile because it extends the Math class, which has been declared as final. A class cannot extend a class that has been declared final.
